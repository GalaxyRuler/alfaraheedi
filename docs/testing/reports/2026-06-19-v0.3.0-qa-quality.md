## Eval And Arabic Rules

Eval changes:

- Added false-negative tracking and recall metrics.
- Missing expected rule sources now produce `missing_expected` failures.
- Expanded the suite from 15 to 22 cases.

RED evidence before rule implementation:

- `cargo run -p write-eval` failed with 3 `missing_expected` failures:
  - `arabic:latin-semicolon`
  - `arabic:space-after-punctuation` twice

GREEN evidence after implementation:

```powershell
cargo test -p write-arabic
cargo run -p write-eval
```

Results:

- `cargo test -p write-arabic`: 7 passed.
- `cargo run -p write-eval`: 22 cases, 13 true positives, 0 false positives, 0 false negatives, precision 1.0, recall 1.0.

## Findings

- S2: Color contrast still needs a manual AA review; automated axe color-contrast was disabled in E2E because rendered font/color precision is noisy without a design-token contrast audit.
- S3: Playwright currently runs Chromium only. Firefox/WebKit should be added once the Chromium lane stays stable.
- S3: The real LLM smoke validates integration and policy, not Arabic model quality. Model quality still needs a separate fixture/eval process before changing the default model.
