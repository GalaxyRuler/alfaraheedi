## VM Packaged DOM/IPC Smoke

To inspect the packaged WebView in the disconnected VM, the app was launched with a temporary WebView2 debug flag:

```text
WEBVIEW2_ADDITIONAL_BROWSER_ARGUMENTS=--remote-debugging-port=9222 --remote-allow-origins=*
```

This was used only inside the VM QA session.

CDP verified the installed packaged DOM at `http://tauri.localhost/`:

- LLM settings section rendered.
- Check runtime button rendered.
- Run doctor button rendered.
- Tauri internals were available in the packaged WebView.
- The VM still had an invalid persisted runtime URL from an earlier policy smoke, so clicking Check runtime showed the Rust-side policy rejection:

```text
ALFARAHEEDI_LLM_BASE_URL must point at a local loopback runtime, for example http://127.0.0.1:8000
```

- Clicking Run doctor opened the runtime doctor panel and showed:

```text
local LLM doctor found blocking configuration issues
```

- The doctor check list included `runtime_config` with outcome `fail` and the same `ALFARAHEEDI_LLM_BASE_URL` loopback hint.
- Directly invoking the packaged Tauri command `cancel_companion_llm_suggestion` with no active local LLM request returned:

```json
{ "ok": true, "value": false }
```

This verifies the packaged frontend can call the Tauri runtime status, doctor, and local LLM cancellation commands, and that the installed Rust host rejects non-loopback LLM runtimes without attempting live model traffic.
