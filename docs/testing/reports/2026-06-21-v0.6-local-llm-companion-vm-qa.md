## VM Packaged DOM/IPC Smoke

To inspect the packaged WebView in the disconnected VM, the app was launched with a temporary WebView2 debug flag:

```text
WEBVIEW2_ADDITIONAL_BROWSER_ARGUMENTS=--remote-debugging-port=9222 --remote-allow-origins=*
```

This was used only inside the VM QA session.

CDP verified the installed packaged DOM at `http://tauri.localhost/`:

- LLM settings section rendered.
- Runtime URL input rendered with empty default.
- Model id input rendered as `qwen3-1.7b-q4_k_m`.
- Timeout input rendered as `30000`.
- Check runtime button rendered.
- Clicking Check runtime with no URL showed:

```text
local LLM runtime is not configured; set ALFARAHEEDI_LLM_BASE_URL
```

CDP then set the runtime URL input to:

```text
https://example.com
```

After clicking Check runtime, the installed app showed the Rust-side policy rejection:

```text
ALFARAHEEDI_LLM_BASE_URL must point at a local loopback runtime, for example http://127.0.0.1:8000
```

This verifies the packaged frontend can call the new Tauri command and that the installed Rust host rejects non-loopback LLM runtimes.
